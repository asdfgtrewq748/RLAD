## 🟡 重要任务（强烈建议 - 显著提升论文质量）

### 任务5：失败案例分析
**审稿人关切**：Reviewer #2 - "建议提供失败案例分析"

**子任务**：
- [ ] 5.1 识别失败案例：
  - 假阴性（FN）：漏报的异常
  - 假阳性（FP）：误报的正常
  - 各选5-10个代表性案例
- [ ] 5.2 分析失败原因：
  - FN：为什么模型没检测到？（特征不明显、与正常模式相似等）
  - FP：为什么模型误判？（噪声干扰、正常但罕见模式等）
- [ ] 5.3 可视化失败案例：
  - 时间序列图
  - 标注真实标签和预测标签
  - 高亮失败区域
- [ ] 5.4 讨论改进方向：
  - 如何减少FN？（更敏感的特征、调整奖励等）
  - 如何减少FP？（更好的去噪、上下文建模等）

**预计时间**：10-12小时

### 任务8：参数选择依据说明
**审稿人关切**：Reviewer #3 - "STL参数、奖励数值缺乏依据"

**子任务**：
- [ ] 8.1 STL参数说明：
  - **季节性周期288**：
    - 解释：采煤班次周期（3个班次 × 8小时 × 12次/小时）
    - 提供领域知识支持
    - 如果有数据，提供频谱分析图
  - **趋势灵活性**：
    - 实验：测试不同值（0.1, 0.5, 1.0, 2.0）
    - 选择效果最好的值
    - 报告实验结果
- [ ] 8.2 LOF参数说明：
  - **阈值 μ + 3σ**：
    - 经验法则（68-95-99.7规则）
    - 实验：对比2σ, 3σ, 4σ
    - 选择平衡precision/recall的值
  - **连续异常数量**：
    - 实验：测试1, 3, 5, 7
    - 选择减少噪声的值
- [ ] 8.3 DQN参数说明：
  - Learning rate, batch size, replay buffer size等
  - 引用DQN文献的推荐值
  - 或报告网格搜索结果
- [ ] 8.4 在方法部分或附录中创建"参数设置"表格

**预计时间**：8-10小时
